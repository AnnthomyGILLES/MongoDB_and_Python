MongoDB 1,2¶

Connexion au serveur¶

Tout d'abord, excuter le client mongo depuis votre terminal. Par dfaut, il se connecte au serveur mongod en localhost sur le port 27017.

In [9]:
client = MongoClient('localhost:27017')
client
Out[9]:
MongoClient(host=['localhost:27017'], document_class=dict, tz_aware=False, connect=True)

Création d'une base de donnée¶

Une fois connect au serveur, il est possible d'accder aux bases de donnes. MongoDB cre des bases et des collections automatiquement si elles n'existent pas dj. Une seule instance MongoDB peut grer de multiples bases de donnes indpendantes. En utilisant PyMongo, on accde ou accde au bases de donnes de la faon suivante:

In [10]:
db = client['GoTerm_database']
db
Out[10]:
Database(MongoClient(host=['localhost:27017'], document_class=dict, tz_aware=False, connect=True), 'GoTerm_database')

Il est possible de connaitre les bases de donnes disponibles en utilisant la mthode database_names().NB: les bases de donnes sans collections ou contenant des collections vides ne seront aps affiches.

In [11]:
client.database_names()
Out[11]:
['GoTerm_database', 'admin', 'config', 'local']

Les collections¶

Une collection est un ensemble de document et peut tre vu comme une table dans une base de donnes relationnelle. Les collections et les bases sont cres de faon "lazy" c'est--dire quand le premier document est insr. Enfin, MongoDB intgre des index nots _id unique pour chaque document.

In [12]:
collection = db['ontology']
db.collection_names()
Out[12]:
['ontology']

Insertion de documents¶

Afin d'insrer un document dans une collection, nous pouvons utiliser la mthode insert_one():

In [99]:
post = res[0]
collection.insert_one(post)
collection

De plus, afin d'insrer une srie de document il est possible d'utiliser la mthode insert_many() qui prend en argument une liste. Cela insrera chaque document contenu dans la liste.

Conditions¶

Les requtes MongoDB sont reprsentes comme une structure JSON. Pour construire une requte, il suffit juste de spcifier un dictionnaire contenant les proprits que nous souhaitons retrouver.

In [194]:
mf=collection.find({"goAspect": "molecular_function"}).count()
cc=collection.find({"goAspect": "cellular_component"}).count()
bp=collection.find({"goAspect": "biological_process"}).count()
print(" Il y a {} GO term annots  Molecular Function, {} Cellular Component et {} Biological Process".format(mf,cc,bp))

 Il y a 98 GO term annots  Molecular Function, 2 Cellular Component et 0 Biological Process

Les opérateurs de requêtes¶

Les requtes peuvent galement faire appel aux oprateurs. Ces oprateurs sont notamment gt, gte, lt, lte, ne, nin, regex, exists, not, or, et bien d'autres . La requte suivante ira extraire tous les GO term antrieur au 11 Juillet 2007.

NB: Nous avons converti les lments dates du json afin d'en faire des objets datetime, aisment manipulable en python

In [197]:
date = datetime.strptime("11/07/2007", "%d/%m/%Y")
_=collection.find({"date": {"$lt":date}}).count()
print("Il y a {} GO term antrieurs au 11 Juillet 2007".format(_))

Il y a 16 GO term antrieurs au 11 Juillet 2007

Aggregation 1,2,3¶

Les oprations d'aggrgations s'excutent directement sur les donnes et retourne un rsultat de type collection. Ces oprations "regroupent" ensembles les valeurs de multiples documents, et peuvent galement procder une large varit d'oprations. Les plus pertinentes sont prsentes ici.

$group¶

Permet de faire un group by SQL et ainsi de retrouver le nombre de X disponibles pour chaque Y.

In [71]:
pipeline = [{"$group": {"_id": "$goAspect", "count": {"$sum": 1}}}]
pprint.pprint(list(collection.aggregate(pipeline)))

[{'_id': 'molecular_function', 'count': 98},
 {'_id': 'cellular_component', 'count': 2}]

$project¶

Permet de remodeler une collection. Dans le cas ci-dessous nous nous contentons de mettre en majuscule tous les identifiants "qualifier" dont le goAspect est cellular_component, mais n'importe qu'elle fonction peut tre applique sur les donnes.

In [95]:
pipeline=[
    { 
    "$match": {
      "goAspect": "cellular_component"
    }
  },
  { "$project":
    { 
      "_id": "$goAspect",
      "qualifier": {"$toUpper": "$qualifier" },
    }
  }
]
pprint.pprint(list(collection.aggregate(pipeline)))

[{'_id': 'cellular_component', 'qualifier': 'COLOCALIZES_WITH'},
 {'_id': 'cellular_component', 'qualifier': 'COLOCALIZES_WITH'}]

Mettre à jour un document d'une collection¶

PyMongo peut mettre jour les documents d'une collection. Pour voir le nombre de documents qui ont match la condition, l'attribut matched_count peut tre utilis sur l'objet. En revanche pour voir le nombre de document mis jour par l'opration, l'attribut modified_count peut tre utilis.

result=collection.update_one({'id': 'UniProtKB:P04637!415151353'},
                     {
                         "$set":{
                             'goName': 'Changement de GoName'
                         }
                         
                     })
result.matched_count
result.modified_count
